# Command: jupyter-dev

## Purpose

Develop Jupyter notebooks against the **NotebookSession API** in `kbutillib.notebook` (post-2026-05 refactor). The legacy `%run util.py` god-class pattern is **deprecated** — current notebooks use a `session = NotebookSession(...)` object with explicit `session.cache`, `session.vectors`, `session.kbu` (KBUtilLib facade), plus per-project pure helper functions in `util.py`.

This skill emphasizes:
- Per-project venvs and template `util.py` bootstrapped by `kbu init-notebook`
- Cells that initialize `session` and use `session.cache.save/load` for intermediate data
- `session.kbu.<sub>.<method>(...)` for KBase / ModelSEED utilities (composition facade)
- Pure helper functions in `util.py` — testable in isolation, never bolted onto a god-class
- No multi-inheritance god-class

## Command Type

`jupyter-dev`

## Input

You will receive a request file containing:
- Notebook development task description
- Project name (folder/repo name; used by `kbu init-notebook`)
- Specific analysis or computation requirements
- Input data files (optional)
- User preferences (optional)

## Project Structure

Notebook projects follow this directory structure (the `kbu init-notebook` CLI generates the bootstrap):

```
<NotebookProject>/
├── notebooks/
│   ├── util.py                  # Slim — generated by `kbu init-notebook` template
│   ├── <notebook-name>.ipynb    # Notebook files (kernel pinned to kbu.nb-<project>)
│   ├── data/                    # Input data (experimental, omics, expression data) — IMMUTABLE
│   ├── genomes/                 # Genome files — IMMUTABLE
│   ├── models/                  # COBRA/COBRApy models — IMMUTABLE
│   ├── nboutput/                # Non-JSON output (TSV, Excel, plots, HTML)
│   └── .kbcache/                # NotebookSession cache (replaces legacy datacache/)
├── pyproject.toml               # Project deps (incl. editable kbutillib + cobrakbase + ModelSEEDpy)
└── README.md
```

**Key rule:** `data/`, `genomes/`, `models/` are **inputs**. They never get regenerated. Only `.kbcache/` is intermediate; if a notebook can't reproduce its outputs from `data/`+`genomes/`+`models/` on a clean run, that's a notebook bug to fix — never a "cache migration" to write.

## util.py Structure (post-refactor)

`util.py` is **slim** — generated by `kbu init-notebook` from `src/kbutillib/cli/templates/util.py.tmpl`. It defines the `session` object and a marker for project-specific helpers:

```python
"""Per-notebook-project utilities.

Generated header (managed by `kbu init-notebook --force`).
Project-specific helpers below the sentinel marker are preserved.
"""
from kbutillib.notebook import NotebookSession
from kbutillib.notebook.helpers import (
    # 10 promoted helpers from the legacy util.py (compartment, reaction, fva, etc.)
    COMPARTMENT_MAP, normalize_compartment,
    get_reaction_directionality, standardize_exchange_id,
    get_exchange_map, build_gene_reaction_map,
    reaction_equation_with_names, is_diffusion_reaction,
    compare_reaction_stoichiometry,
    find_significant_differences, classify_fva_flux,
)

# Module-level session — every cell uses this
session = NotebookSession(
    name="<PROJECT_NAME>",
    notebook_folder=__file__.rsplit("/", 1)[0],
)


def session_for(file: str) -> NotebookSession:
    """Back-compat shim for cells that pass __file__."""
    return session


# === project-specific helpers below ===
# Add pure helper functions here. Test them in test_util.py.
# Functions here are FREE FUNCTIONS, never methods on a god-class.

def my_project_helper(model, threshold=0.1):
    """Pure helper — testable in isolation."""
    ...
```

### Key Points for util.py

1. **Generated by `kbu init-notebook`** — don't hand-author the header above the marker. To regenerate: `kbu init-notebook --force` (preserves code below the sentinel).
2. **Project-specific helpers go below the sentinel** as **pure free functions** — never methods on a god-class. Testable in `tests/test_util.py`.
3. **`session` is module-level** — every cell uses the same instance.
4. **`session.kbu` is the KBUtilLib facade** — for any KBase/ModelSEED utility (`session.kbu.fba.run_fba(model)`, `session.kbu.biochem.search_compounds("glc")`, etc.). NOT the legacy `NotebookUtils` god-class (which was deleted).

## Notebook Cell Design Pattern

Every notebook cell follows this pattern:

### 1. Markdown Cell (Always First)
```markdown
## [Step Name/Purpose]

[Explanation of what this code cell does and why]
- Key objective
- Input data used
- Output data produced
- Any important notes
```

### 2. Code Cell (Always Second)
```python
from util import session

# Load required data from previous steps (or from data/ on first cell)
data1 = session.cache.load("data1_name")
data2 = session.cache.load("data2_name")

# Perform analysis/computation
result = some_analysis(data1, data2)

# Save intermediate results for cell independence
session.cache.save("result_name", result)
```

### Critical Cell Design Rules

1. **Every code cell imports session**: `from util import session`
   - Replaces the legacy `%run util.py` magic
   - Module-level singleton — same env, same cache across cells

2. **Load via `session.cache.load(name)`** for any intermediate data
   - Reads from `.kbcache/` (project-wide flat namespace; no per-notebook subdirs)
   - Returns the saved object (JSON/Pickle/parquet — engine handles type)

3. **Save via `session.cache.save(name, data)`** for outputs
   - Writes to `.kbcache/`
   - For non-JSON output (TSV/Excel/HTML/PNG), write to `nboutput/` directly with the file API

4. **Cell independence**: Each cell should run on its own
   - Don't rely on variables from previous cells without loading them via `session.cache.load`
   - Don't assume cells run in order
   - Re-running a single cell should work

5. **Markdown precedes code**: Every code cell has a markdown cell explaining it

### KBase/ModelSEED utilities go through `session.kbu`

```python
# Cell: build a metabolic model from a genome
from util import session

genome = session.cache.load("ADP1Genome")  # cached earlier
model = session.kbu.recon.build_model_from_genome(genome, template="GramNegative")
session.cache.save("ADP1Model_draft", model)
```

```python
# Cell: run FBA (preserves AP3 carve-outs from MSFBAUtilsImpl)
from util import session

model = session.cache.load("ADP1Model_draft")
session.kbu.fba.set_media(model, "Complete")
solution = session.kbu.fba.run_fba(model, biomass_reaction="bio1")
fva = session.kbu.fba.run_fva(model)  # AP3 carve-out — working FVA
session.cache.save("FBA_solution", solution)
session.cache.save("FVA_results", fva)
```

```python
# Cell: AI curation
from util import session

reactions = session.kbu.biochem.search_reactions("transport")
for rxn in reactions[:10]:
    direction = session.kbu.curation.curate_reaction_direction(rxn)
    session.cache.save(f"curation_{rxn['id']}", direction)
```

## Process

### Phase 1: Bootstrap the Project

1. **Run `kbu init-notebook` once per machine + project**
   ```bash
   cd <NotebookProject>
   kbu init-notebook
   ```
   This will:
   - Resolve machine alias from `~/.agentforge/config.yaml` or hardware UUID
   - Create a per-project venv via `venvman create --project kbu.nb-<project>`
   - `pip install -e` the sibling repos (KBUtilLib, cobrakbase, ModelSEEDpy)
   - Generate `notebooks/util.py` from the template (with NotebookSession-based code)
   - Register a Jupyter kernel `kbu.nb-<project>`
   - Pin all `*.ipynb` files to that kernel

2. **Verify Required Subdirectories**
   - `notebooks/data/`, `notebooks/genomes/`, `notebooks/models/` for inputs
   - `notebooks/nboutput/` for non-JSON output
   - `notebooks/.kbcache/` is auto-created by NotebookSession

3. **Smart-merge if util.py exists**
   - `kbu init-notebook --force` smart-merges the template header above the `# === project-specific helpers below ===` sentinel; preserves all custom code below.
   - Without `--force`, existing `util.py` is never touched.

### Phase 2: Understand Requirements

4. **Analyze Task Description**
   - Identify the scientific/analytical goal
   - Determine required input data (which `data/`/`genomes/`/`models/` files)
   - Identify computation steps and `session.kbu.*` calls needed
   - Plan logical cell breakdown

5. **Plan Notebook Structure**
   - Break task into logical steps (cells)
   - Identify cache keys for inter-cell data flow
   - Determine which `session.kbu.*` sub-utilities are needed
   - Plan any project-specific helpers for `util.py`

### Phase 3: Develop Project-Specific Helpers (only if needed)

6. **Add helpers to util.py BELOW the sentinel marker**
   - Add only **pure free functions** (no inheritance!)
   - One responsibility each, testable in isolation
   - Add corresponding tests in `tests/test_util.py`
   - **Audit rule**: any new helper whose body is more than a few lines is suspect — make sure it's not silently reimplementing a `session.kbu.*` method (e.g., FBA, set_media, Escher map building). Always prefer `session.kbu.*` for canonical operations.

### Phase 4: Create/Modify Notebook

7. **Create Notebook Cells**
   - For each logical step:
     - Markdown cell explaining the step
     - Code cell with proper pattern: `from util import session`, load via `session.cache.load`, compute, save via `session.cache.save`
   - Use `session.kbu.<sub>.<method>(...)` for KBase/ModelSEED ops
   - Each cell independent and re-runnable

8. **Organize Data Files**
   - Inputs in `data/`, `genomes/`, `models/` (immutable)
   - Non-JSON output to `nboutput/`
   - `session.cache.save()` handles `.kbcache/` automatically

### Phase 5: Validate and Document

9. **Verify Notebook Standards**
   - Every code cell starts with `from util import session`
   - Every code cell has preceding markdown
   - Data dependencies use `session.cache.load`
   - Results saved with `session.cache.save`
   - Cells can run independently
   - All files in correct directories
   - **No `%run util.py` legacy magic**
   - **No `class XxxTools(A, B): pass` multi-inheritance**

10. **Create Summary Documentation**
    - Notebook purpose and workflow
    - Required input data and locations
    - Each major step
    - Manual setup notes

### Phase 6: Save Structured Output

11. **Save JSON Tracking File** (see schema below)

## Antipatterns to avoid (observed during 2026-05 migration)

These came up repeatedly during the BERDL-trio migration. Pre-flight self-check before delivering a notebook:

1. **Hardcoded `GROWTH_DASH_RXN`** — Legacy ModelSEED biomass id. Use `bio1` instead and pass `biomass_reaction="bio1"` explicitly to `session.kbu.fba.run_fba(model, biomass_reaction="bio1")`. Symptom: `status="optimal"` but `biomass=0`.

2. **Missing `set_media`** — Don't drop media application before FBA. Use `session.kbu.fba.set_media(model, media)` (or `session.kbu.fba.run_fba(model, media=...)`). Symptom: model has no carbon source bounded → biomass=0.

3. **Reimplementing canonical methods as heuristics** — If you write a `def run_X(...)` in util.py whose body is more than a few lines, verify it's not silently reinventing `session.kbu.fba.*`, `session.kbu.escher.*`, `MSExpression`, etc. Prefer the canonical method through the facade.

4. **Slash-prefixed cache keys for cross-notebook reads** — Project-wide cache is flat. If notebook A saves `"foo"` and notebook B loads `"<NotebookA>/foo"`, B will KeyError. Strip the prefix: load `"foo"` instead.

5. **`sol.fluxes.values()` on a pandas Series** — `.values` is an attribute, not a method. Use `(sol.fluxes.abs() > 1e-9).sum()` or iterate the Series directly.

## Migration from legacy notebooks

If you find a notebook using the legacy pattern (`%run util.py`, `tools = MyTools()`, multi-inheritance god-class):

**Recommended recipe (per the 2026-05 PRD):**
1. **Archive** the legacy notebook(s) to `notebooks/archive/` rather than migrating in-place. Don't try to mechanically translate cell-by-cell.
2. **Rebuild collaboratively** with the user against the new `session` API, using the archived code as reference.
3. The legacy `util_legacy.py` shim is no longer needed once the rebuild is complete.

For the full migration playbook used during the KBUtilLib refactor, see `KBUtilLib/agent-io/docs/notebook-migration-playbook.md`.

## Design Principles

### Cell Independence Philosophy

Cell independence enables:
1. Re-run individual cells without executing the whole notebook
2. Skip expensive computations by loading cached results
3. Recover from failures without losing progress
4. Test variations by modifying single cells
5. Collaborate (others understand and modify individual steps)

Implementation:
- `session.cache.save/load` creates checkpoints in `.kbcache/`
- `from util import session` ensures consistent environment
- Markdown cells provide context

### When to Cache

Cache when:
- Results took significant time to compute
- Data will be used by multiple subsequent cells
- Intermediate results are worth preserving
- Enabling cell re-runs would save time

Don't cache when:
- Quick computations (< 1 second)
- Data only used in next cell
- Data is not JSON/pickle-serializable (write to `nboutput/` instead)

## Error Handling

1. **`kbu init-notebook` fails** — most common: Python version (use 3.11; scientific Python ecosystem incompat with 3.13+). Note in errors and suggest `--python 3.11`.
2. **Existing util.py with no sentinel** — refuse `--force`; ask user to manually insert the `# === project-specific helpers below ===` marker.
3. **Existing util.py with sentinel** — `--force` smart-merges header, preserves below-sentinel code.
4. **Non-JSON Data** — guide user to write to `nboutput/` directly.
5. **Long-Running Cells** — emphasize `session.cache.save` for intermediate results.

## Privacy and Security Considerations

- Don't include API keys or credentials in `util.py` or notebooks
- Tokens come from `session.kbu.env.get_token(...)`, which reads `~/.kbase/token` and env vars
- Don't log sensitive data into `.kbcache/`
- Note if data files contain sensitive information

## Example Workflow

For a typical metabolic modeling notebook:

1. **Cell 1 (markdown + code)**: Load genome from `genomes/`
   ```python
   from util import session
   genome = session.kbu.genome.get_genome_from_file("genomes/ADP1.gff")
   session.cache.save("ADP1Genome", genome)
   ```

2. **Cell 2**: Load COBRA model from `models/`
   ```python
   from util import session
   import cobra
   model = cobra.io.load_json_model("models/MergedADP1Model.json")
   session.cache.save("ADP1Model", model)
   ```

3. **Cell 3**: Apply media + run FBA
   ```python
   from util import session
   model = session.cache.load("ADP1Model")
   session.kbu.fba.set_media(model, "Complete")
   solution = session.kbu.fba.run_fba(model, biomass_reaction="bio1")
   session.cache.save("FBA_solution", solution)
   ```

4. **Cell 4**: FVA + analysis (AP3 carve-out — uses kbu.fba.run_fva, NOT cobra)
   ```python
   from util import session
   model = session.cache.load("ADP1Model")
   session.kbu.fba.set_fraction_of_optimum(model, 0.9)
   fva = session.kbu.fba.run_fva(model)
   session.cache.save("FVA_results", fva)
   ```

5. **Cell 5**: Generate Escher map → `nboutput/`
   ```python
   from util import session
   import os
   model = session.cache.load("ADP1Model")
   solution = session.cache.load("FBA_solution")
   html = session.kbu.escher.create_map_html2(model, solution.fluxes)
   os.makedirs("nboutput", exist_ok=True)
   with open("nboutput/flux_map.html", "w") as f: f.write(html)
   ```

Each cell independent, each with clear purpose, each properly cached. KBase/ModelSEED utilities accessed via `session.kbu.*`. No multi-inheritance, no god-class, no `%run util.py`.
